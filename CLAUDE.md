# CLAUDE.md

Это руководство для Claude при работе с репозиторием **MEbel** — сайта мебельной компании.

## Обзор проекта

Лендинг компании MEbel (мебель и интерьер на заказ). Показывает портфолио, направления работы (кухни, шкафы, шторы, жалюзи), процесс работы, отзывы и форму обратной связи с отправкой в Telegram.

**Стек:** React 18 + TypeScript + Vite + Tailwind CSS + shadcn-ui
**Деплой:** Netlify (автоматически из ветки `main`)

## Команды разработки

```bash
npm run dev          # Дев-сервер (http://localhost:8080)
npm run build        # Продакшн-сборка в dist/
npm run build:dev    # Сборка в dev-режиме
npm run preview      # Превью продакшн-сборки
npm run lint         # ESLint проверка
npm run test         # Запуск unit-тестов (Vitest, однократно)
npm run test:watch   # Тесты в watch-режиме
```

> E2E-тесты (Playwright) запускаются отдельно через `npx playwright test`.

## Соглашения по коду

### TypeScript
- Строгая типизация обязательна — никаких `any` без крайней необходимости
- Использовать Zod для валидации форм и внешних данных
- Экспортировать типы через `export type`, не `export`

### Компоненты
- Функциональные компоненты с TypeScript, без class-компонентов
- Один компонент — один файл; имя файла совпадает с именем компонента
- `shadcn-ui` компоненты в `src/components/ui/` — не изменять вручную, добавлять через `npx shadcn-ui@latest add <component>`
- Использовать хук `useIsMobile()` для адаптивной логики

### Стилизация
- **Только Tailwind CSS** — никаких inline-стилей и отдельных CSS-файлов кроме `index.css`
- Использовать CSS-переменные из темы (`--primary`, `--background`, и т.д.), не хардкодить цвета
- Тёмная тема управляется через `next-themes` и класс `dark` на `<html>`
- Утилита `cn()` из `@/lib/utils` для условных классов

### Импорты
- Псевдоним `@/` указывает на `src/` — использовать его вместо относительных путей глубже одного уровня
- Порядок: внешние библиотеки → внутренние модули (`@/`) → относительные

### Формы
- React Hook Form + Zod-схема для всех форм
- Компоненты `Form`, `FormField`, `FormItem` из shadcn-ui

## Работа с UI-компонентами

Добавление нового shadcn-ui компонента:
```bash
npx shadcn-ui@latest add <component-name>
```

Компоненты автоматически попадают в `src/components/ui/`.

## Анимации

Framer Motion (`framer-motion`) — для анимаций появления и переходов. Использовать компонент `AnimatedSection` для стандартного появления секций при скролле.

## Тестирование

- **Юнит-тесты:** Vitest + Testing Library, файлы рядом с компонентом или в `src/test/`
- **E2E:** Playwright, конфиг в `playwright.config.ts`
- Тестировать логику и взаимодействия, не реализацию

## Деплой и окружение

- Деплой на **Netlify** автоматически при пуше в `main`
- Node.js версия: **20** (задана в `netlify.toml`)
- SPA-маршрутизация через `_redirects` или настройки Netlify
- Переменные окружения задаются в панели Netlify, к коду не привязываются
- Для Telegram-уведомлений используется Netlify Forms + serverless function

## Что не делать

- Не редактировать файлы в `src/components/ui/` вручную
- Не добавлять CSS-файлы к отдельным компонентам — только Tailwind
- Не использовать `useState` для серверного состояния — для этого TanStack Query
- Не коммитить `.env`-файлы и секреты
- Не менять структуру `tsconfig.json` без необходимости
